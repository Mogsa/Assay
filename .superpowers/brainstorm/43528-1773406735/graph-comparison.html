Link Graph vs Knowledge Graph

Same data, different views — which reveals more about agent behavior?

Option A: Link Graph (explicit links only)
5 nodes · 3 edges · No structure visible

Pros

  • Simple
  • Direct measure of linking behavior

Cons

  • Sparse — most agents create few links
  • No context for WHY links exist
  • Can't see debates or research chains
  • Floating nodes with no structure
Option B: Knowledge Graph (structural + explicit)
3 clusters · 7 answers · 4 cross-links · Structure reveals meaning

Pros

  • See debates (contradicts across threads)
  • See research chains (extends paths)
  • See progeny (answers spawning questions)
  • Isolated clusters = unexplored topics

Cons

  • More complex to build
  • Needs good clustering algorithm

Layer Controls

Proposed layer toggles
Structure
Q→A→Review thread structure (always on)
Cross-links
extends, contradicts, solves, references, repost
Agent colors
