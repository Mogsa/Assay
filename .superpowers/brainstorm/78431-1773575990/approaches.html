Knowledge Graph Redesign — 3 Approaches

How should we restructure the visualization to make connections clear?

A

Community Clusters + Semantic Overlay

Core idea: Group questions by community in distinct visual clusters. Hide structural edges (Q→A→C) by default — collapse each question into a single node showing its answer/review count as a badge. Only show semantic edges (extends, contradicts, references, solves) as the connecting lines.

Cross-community links become dramatic arcs between clusters — the most visually prominent element.

Frontier Map becomes a per-community view: select a community, see its questions classified as frontier/explored/debated.

Pros

  • Drops visible nodes from 630 → ~124 questions
  • Semantic links become the star, not noise
  • Community structure is immediately obvious
  • Cross-community connections stand out

Cons

  • Loses per-answer/review detail at top level (need click-to-expand)
  • Most work — restructures both views
B

Community Tabs + Better Current Graph

Core idea: Add a community selector dropdown. Each community shows its own force graph with all nodes (questions, answers, reviews). Fix the current issues: thicker structural lines, working sidebar filters, hover tooltips, fix detail panel bug.

Cross-community links shown as a separate "Cross-community" tab that only displays questions with inter-community semantic links.

Frontier Map stays as-is but scoped per community (fewer nodes = less blob).

Pros

  • Least code change — mostly fixes + a dropdown
  • Keeps answer/review detail visible
  • Familiar layout, just less overwhelming

Cons

  • Still shows structural edges as the dominant visual
  • Question-to-question connections still buried
  • Cross-community view is a separate tab, not integrated
C

Question Map (semantic only) + Drill-down

Core idea: The default view ONLY shows questions as nodes, with ONLY semantic edges (extends, contradicts, references, solves). Communities shown as colored halos/backgrounds behind their question clusters. Click a question to expand it and see its answers and reviews in a detail panel.

Cross-community links are just normal edges — they naturally span between community halos, making them visually obvious.

Frontier Map replaced: frontier/debated/isolated status becomes a color or icon on the question nodes in the main map (toggle-able layer).

Pros

  • Maximum clarity — every visible line is a meaningful connection
  • Only ~124 nodes + ~60 semantic edges = clean, readable graph
  • Cross-community links are naturally prominent
  • Merges two views into one (simpler UX)
  • Frontier status visible without switching tabs

Cons

  • Medium effort — new default view, detail panel rework
  • Loses the "see everything" force graph (could keep as advanced toggle)

My recommendation: C. It directly solves "make question-to-question connections clear" and "reduce overwhelm" in one move. The current full graph could stay as an "Advanced" toggle for when you want the detail.
